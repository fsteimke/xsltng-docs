Anhang B. Versionshistorie

B.1. Änderungen gegenüber XUnternehmen.Basismodul 1.1

Dieser Abschnitt führt alle nicht rein redaktionellen Änderungen der Version 1.2 des Standards XUnternehmen.Basismodul gegenüber der Version 1.1 auf.

Für weiterführende Informationen insbesondere zur Begründung Änderung wird auf das jeweilige Change Request Ticket (KDM-XY) verwiesen.

B.1.1. Nachvollzogene Änderungen aus XUnternehmen.Kerndatenmodell 1.2

Die folgenden Änderungen im Standard XUnternehmen.Kerndatenmodell 1.2 wurden im Standard XUnternehmen.Basismodul 1.2 eins-zu-eins nachvollzogen.

KDM-141: Auch Wirtschaftliche Tätigkeit in der Rolle Antragsteller/Anzeigender zulassen

Eine neue Codeliste urn:xoev-de:xunternehmen:codeliste:angabeantragstelleranzeigender und ein neuer Code-Datentyp antr:Code.ArtAngabeAntragstellerAnzeigender wurden angelegt.

Das Elemente antr:antragsteller wurde um ein Kindelement antr:artAngabeAntragsteller (Typ antr:Code.ArtAngabeAntragstellerAnzeigender) ergänzt; das Kindelement antr:antragsteller.id wurde optional gemacht.

Das Elemente antr:anzeigender wurde um ein Kindelement antr:artAngabeAnzeigender (Typ antr:Code.ArtAngabeAntragstellerAnzeigender) ergänzt; das Kindelement antr:anzeigender.id wurde optional gemacht.

KDM-149: Metadaten zum WZ-Schlüssel ergänzen

Ein neues Element wt:wirtschaftszweig wurde eingeführt. Das Element wt:wirtschaftszweigschluessel wird nun als Kindelement des neuen Elements verwendet und es wurde der Bezug zur Version WZ-2008 entfernt. Ein zweites Kindelement ermöglicht die Angabe der Version der Klassifikation der Wirtschaftszweige als Code, bspw. "WZ-2008" oder "WZ-2025".

Dazu wurde eine neue Codeliste urn:xoev-de:xunternehmen:codeliste:wirtschaftszweigsklassifikationsversion bereitgestellt.

KDM-159: Klarstellung zum Attribut Eintragungsnummer in Registereintrag (Ortskennzeichen)

Die Dokumentation des Elements eintr:eintragungsnummer wurde dahingehend ergänzt, dass hierunter auch ein etwaiges Ortskennzeichen fällt.

KDM-165: Bedeutung von Geschäftsbezeichnung

Die Dokumentation des Kerndatenobjekts "Geschäftsbezeichnung" wurde dahingehend angepasst, dass die Geschäftsbezeichnung von einem ggf. im Justizregister eingetragenen Namen abweichen kann, dies aber nicht muss.

KDM-164: Eintragungsart "Kraft Gesetz" streichen

Die Beschreibung der Elemente ename:eingetragenerName und eintr:eintragung wurde dahingehend enger gefasst, dass nur der in einem Justizregister eingetragene Name hierunter gefasst wird, nicht aber (= gestrichen) der Name, der kraft Gesetz besteht. Die Eintragungsart "Kraft Gesetz" wurde entsprechend aus der Codeliste "Art der Eintragung" entfernt und die Beschreibung der Codeliste enger gefasst als Eintragung in einem *Justiz*register.

KDM-168: Aufnahme eines Elements für spezifische Zusatzinformationen zur flexibleren Reaktion auf kurzfristige Ergänzungsbedarfe

Aufnahme eines neuen globalen Elements spezifischeInformation mit Kindelementen begriff und inhalt (beides Zeichenkette).

Erweiterung der Ausprägungen der Kerndatenobjekte "Antrag eines Wirtschaftlich Tätigen" und "Anzeige eines Wirtschaftlich Tätigen" mit einer Referenz auf spezifischeInformation, Anzahl 0...n.

KDM-178: Umstellung von DIN SPEC 91379 auf DIN 91379

Der Datentyp bdt:String.91379.C wurde von der DIN SPEC 91379 auf die DIN 91379:2022-08 umgestellt.

KDM-183: Gesetzliche Vertreter aus der rechtsfähigen Personengesellschaft entfernen

Das Element gv:gesetzlicherVertreter wurde aus dem Element rpg:rechtsfaehigePersonengesellschaft gestrichen.

Die Dokumentation des Elements gv:gesetzlicherVertreter wurde dahingehend angepasst, dass Personengesellschaften nicht als gesetzliche Vertreter auftreten.

Die Dokumentation des Elements gv:gesellschafter wurde dahingehend angepasst, dass Personengesellschaften nicht als gesetzliche Vertreter auftreten.

KDM-184: "Sonstiger Personenvereinigung" hat "Beteiligter" statt "Gesetzlicher Vertreter"

Eine neues Schema (urn:xoev-de:xunternehmen:standard:basismodul:beteiligtersonstigepersonenvereinigung) mit einem neuen Element bpsv:beteiligter.id für einen Beteiligten einer sonstigen Personenvereinigung wurde aufgenommen.

In spv:sonstigePersonenvereinigung wurde das Element gv:gesetzlicherVertreter durch bspv:beteiligter.id ersetzt.

Die Dokumentation des Elements gv:gesetzlicherVertreter wurde dahingehend angepasst, dass sonstige Personenvereinigungen nicht als gesetzliche Vertreter auftreten.

Die Dokumentation des Elements gv:gesellschafter wurde dahingehend angepasst, dass sonstige Personenvereinigungen nicht als gesetzliche Vertreter auftreten.

KDM-188: Redaktionelle Anpassung: Darstellung der FIM-IDs in der Spezifikation

Im Rahmen der Überarbeitung der Nachveröffentlichung des Standards im Baukasten BOB des Föderalen Informationsmanagements wurden einige abstrakte Datenfeldgruppen zugunsten harmonisierter Datenfeldgruppen aufgelöst. Entsprechend wurden bei einigen Elementen und Datentypen die dargestellten (abstrakten) FIM-IDs entfernt.

B.1.2. XÖV-spezifische Änderungen

Die nachfolgenden Änderungen sind XÖV-spezifisch, d.h. sie sind ohne entsprechende Änderung in XUnternehmen.Kerndatenmodell 1.2.

KDM-168 Basismodul: Aufnahme eines Elements für spezifische Zusatzinformationen zur flexibleren Reaktion auf kurzfristige Ergänzungsbedarfe

Um flexibel auf zwingende kurzfristige Ergänzungsbedarfe reagieren zu können, wurde im Basismodul für Antrags-/Anzeige-Nachrichten ein Element antr:spezifischeInformation aufgenommen. Dieses kann technisch mehrfach vorkommen und enthält die Kindelemente antr:begriff und antr:inhalt.

KDM-169: Dokumentation der Elemente uuid und zeitstempel anpassen

Die Dokumentation des Elements antr:uuid und antr:zeitstempel wurde generischer gefasst, sodass die Eingrenzung auf Anträge und Anzeigen entfällt.
